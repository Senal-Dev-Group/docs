---
title: "派生 API Key"
api: "POST /v1/mm/auth/derive-api-key"
description: "派生 API Key：已有则返回现有，无则创建新的（需 L1 EIP-712 认证）"
---

与 [Create API Key](/api-reference/auth/create-api-key) 的区别：

- **Create**：每次调用都创建新 Key
- **Derive**：若当前钱包已有 API Key，返回现有 Key；若没有，则创建并返回

适合幂等初始化场景：无论调用多少次，同一钱包只会对应一个 API Key。

<Note>
  此端点使用 **L1 EIP-712** 认证，详见 [Authentication](/authentication)。
</Note>

## 请求头

<ParamField header="SENAL-ADDRESS" type="string" required>
  钱包地址（EIP-55 checksum 格式），例如 `0xAbCd...`
</ParamField>

<ParamField header="SENAL-SIGNATURE" type="string" required>
  EIP-712 签名结果（hex 字符串）
</ParamField>

<ParamField header="SENAL-TIMESTAMP" type="string" required>
  当前 Unix 时间戳（**秒**），必须在服务器时间 ±60 秒内
</ParamField>

<ParamField header="SENAL-NONCE" type="string" required>
  随机字符串，防重放攻击
</ParamField>

## 响应

<ResponseField name="id" type="string">
  API Key 记录 ID
</ResponseField>

<ResponseField name="apiKey" type="string">
  API Key（请求时放入 SENAL-API-KEY Header）
</ResponseField>

<ResponseField name="apiSecret" type="string">
  API Secret（用于计算 HMAC-SHA256 签名，**仅在首次创建时返回**）
</ResponseField>

<ResponseField name="passphrase" type="string">
  Passphrase（**仅在首次创建时返回**）
</ResponseField>

<Warning>
  如果 Key 已存在，`apiSecret` 和 `passphrase` 将**不会**在响应中返回（出于安全考虑）。若遗失，需联系管理员重置。
</Warning>

## 请求示例

```bash
curl -X POST https://api.senal.market/v1/mm/auth/derive-api-key \
  -H "SENAL-ADDRESS: 0xYourWalletAddress" \
  -H "SENAL-SIGNATURE: 0xeip712signature..." \
  -H "SENAL-TIMESTAMP: 1735689600" \
  -H "SENAL-NONCE: a1b2c3d4e5f6g7h8"
```

## 响应示例

首次派生（创建新 Key）：

```json
{
  "id": "key_01HXYZ123456",
  "apiKey": "ak_abcdef1234567890abcdef1234567890",
  "apiSecret": "sk_9876543210fedcba9876543210fedcba",
  "passphrase": "pp_randomPassphraseString123"
}
```

已存在时返回（`apiSecret` 和 `passphrase` 不返回）：

```json
{
  "id": "key_01HXYZ123456",
  "apiKey": "ak_abcdef1234567890abcdef1234567890"
}
```
